## System Overview

This repository implements a **cooperative multi-agent pipeline** designed to execute natural language queries against uploaded CSV datasets. Rather than routing complex tasks to a single LLM prompt wrapper, this architecture relies on a **state-graph pipeline** implemented in LangGraph. 

The system maps queries to structured execution plans, generates deterministic code paths, executes those paths inside restricted execution environments, and applies a self-correcting compiler loop on compilation or runtime exceptions.

## Architectural Highlights & Engineering Patterns

### 1. LangGraph State Machine & Conditional Routing
The pipeline maintains a central, mutable state (`AnalysisState` TypedDict) representing the session context. Nodes are decoupled and isolated; they read parameters from the state and write back partial state transitions. Edge routing is dynamically determined by the execution status:
- On successful code execution, the state routes directly to the **Insight Agent** for final natural language generation (NLG).
- On execution error, the state transition diagram routes to the **Fixer Agent**, passing prior code blocks, stack traces, and column profiles to iteratively heal the script.

### 2. Runtime Sandboxing & Execution Isolation
To securely run LLM-generated Python code on local infrastructure, execution is constrained inside a dual-engine execution sandbox:
- **Restricted Global Namespace:** Blocks standard built-in entry points (`open`, `eval`, `exec`, `compile`, `__import__`).
- **Memory Mutation Protections (`ReadOnlyProxy`):** Wraps data libraries (`pandas`, `numpy`, `plotly.express`) in dynamic read-only proxies. Any code attempt to mutate library properties (such as overwriting `px.pie`) fails deterministically with an `AttributeError` at runtime, preserving the singleton space of the application.
- **Thread Timeout Boundaries:** Offloads calculations to a background daemon thread with a strict 10-second termination limit to avoid thread locks from infinite loops.
- **In-Memory SQL Execution Sandbox:** When raw SQL is requested, the system loads the target DataFrame into an ephemeral, in-memory SQLite database, validates the AST (restricting queries strictly to `SELECT` operations), and terminates the engine immediately post-fetch.

### 3. State-Aware Error Correction (Self-Healing Compiler)
Rather than simple retries, the **Fixer Agent** behaves like a localized compiler. It evaluates:
1. The currently failing code segment.
2. The runtime exception type (e.g. `KeyError`, `TypeError`, `sqlite3.OperationalError`).
3. Rich context payloads automatically generated by the sandbox (e.g., actual schema column names on `KeyError` or column data types on `TypeError`).
4. An index of previous failing variations to prevent loop convergence on the same error.

| **Execution Layer** | [execution_layer.py](file:///d:/llm%20orchestration/orchestrator/nodes/execution_layer.py) | Heuristic | Directs the code string into either the `safe_exec` boundary or the SQLite virtual memory loop. |

## Technical Specifications

- **Orchestration Layer:** LangGraph State Machine.
- **Inference Engines:** NVIDIA Build API:
  - *Primary:* `nvidia/nemotron-3-ultra-550b-a55b` (Nemotron 3 Ultra 550B).
  - *Fallback:* `nvidia/nemotron-3-super-120b-a12b` (Nemotron 3 Super 120B).
- **Execution Boundaries:** Virtual namespace Python `exec()`, `sqlite3` in-memory structures.
- **Telemetry & Auditing:** Embedded SQLite engine logging complete transaction traces.

## Setup & Deployment

### Prerequisites
- Python **3.9+**
- Active **NVIDIA Build API Key** (Provisioned via [build.nvidia.com](https://build.nvidia.com/))

### Installation & Initialization
```bash
# 1. Clone local workspace
git clone https://github.com/Prajwallnaik/MultiAgent_DataAnalyst.git
cd MultiAgent_DataAnalyst

# 2. Build Python runtime dependencies
pip install -r requirements.txt
```

### Credentials Setup
Configure the environment file in your project root directory:
```env
NVIDIA_API_KEY=nvapi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Start the Application
Execute the Streamlit application runner:
```bash
python -m streamlit run app.py
```
